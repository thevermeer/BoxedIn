BoxedIn

Rule IDs: bundled static rules in rules.json use 1–9999 (currently 1–8). Your custom rules use dynamic IDs 10000–10049. See rules-ids.txt in the extension folder when merging changes.

A small BoxedIn blocks panel appears at the bottom-left of every page with cumulative block counts per rule (requires the declarativeNetRequestFeedback permission so Chrome can report which rules matched). With hostname capture enabled, the panel lists every hostname contacted by the page; check a hostname to block it. Page-guard also reports fetch/XHR/sendBeacon blocks for the LinkedIn URL blocklist and separate counts for chrome-extension://… probes (JS layer; stored as pageGuardBlockStats). Counts are stored in this browser.

Bundled rules

Shipped in rules.json. LinkedIn changes URLs over time—if something breaks, compare this table to the file and update both.

ID What (urlFilter) Why Risk if removed
1 *linkedin.com/sensorCollect* Blocks LinkedIn sensor/collect telemetry requests. More sensor/collection traffic may reach LinkedIn.
2 *chrome-extension://invalid* Blocks extension-scheme probe URLs used by fingerprinting scripts. Sites may probe chrome-extension:// more effectively.
3 *linkedin.com/li/track* Blocks LinkedIn tracking / beacon endpoints. Tracking and analytics requests to these paths may succeed.
4 *collector-pxdojv695v.protechts.net* Blocks Protechts third-party collector traffic. That collector domain may receive requests again.
5 *linkedin.com/realtime/realtimeFrontendClientConnectivityTracking* Blocks realtime frontend connectivity tracking calls. Extra connectivity / client tracking pings may fire.
6 *linkedin.com/tscp-serving* Blocks TSCP serving (ads / related content). More TSCP-serving content or requests may load.
7 *cs.ns1p.net* Blocks NS1-related third-party requests (often ad/telemetry). Traffic to that host may resume.
8 *linkedin.com/li/tscp/sct* Blocks POSTs to LinkedIn TSCP SCT paths. Those POSTs may succeed again (more telemetry / serving).

Blocked hostnames

Hostnames checked in the overlay. All sub-resource requests (scripts, images, XHR, etc.) to these hosts are blocked via dynamic DNR rules (IDs 20000+). Uncheck in the overlay or remove here.

    Custom network block rules

    Extra declarativeNetRequest urlFilter patterns (block requests matching any). Bundled rules above are unchanged; these are added as dynamic rules (IDs 10000+). Patterns are validated before save (length, wildcards, line breaks). Apply status appears below after the service worker runs.

      Red-team tools

      Passive security analysis tools for red-team exercises. When enabled, the overlay gains Auth, Exfil, and Inject tabs with live findings. All analysis is local and passive — no traffic is modified.

      Exfiltration allowlist

      Hostnames in this list will not trigger exfiltration alerts (e.g. your own API domains). The page's own origin is always allowed.
